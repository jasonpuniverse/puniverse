# Project Constitution (B.L.A.S.T Protocol)

## Behavioral Rules
- Prioritize reliability over speed.
- Never guess at business logic.
- Data-First rule strictly enforced.
- Self-Annealing rule for error handling.

## Architectural Invariants
- 3-Layer Architecture (SOPs, Navigation, Tools).
- Business logic in Layer 1 (`architecture/`).
- Probabilistic reasoning in Layer 2 (Navigation).
- Deterministic scripts in Layer 3 (`tools/`).
