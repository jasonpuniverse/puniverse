# Nano Banana Pro 画像生成スキル

## 概要

このスキルは Google Nano Banana Pro API を使用して画像を生成します。
ユーザーが画像生成を必要とする場面で、このスキルを使って画像を作成できます。

## 前提条件

- 環境変数 `CCSKILL_NANOBANANA_DIR` にこのスキルのリポジトリパスを設定してください
  ```bash
  export CCSKILL_NANOBANANA_DIR="$HOME/projects/ccskill-nanobanana"
  ```
- 環境変数 `GEMINI_API_KEY` が設定されている必要があります（または `$CCSKILL_NANOBANANA_DIR/.env` に記載）

## 使い方

以下のコマンドで画像を生成します：

```bash
$CCSKILL_NANOBANANA_DIR/venv/bin/python $CCSKILL_NANOBANANA_DIR/generate_image.py "プロンプト"
```

### オプション

- `--resolution`: 解像度（1K, 2K, 4K）デフォルト: 2K
- `--aspect`: アスペクト比（1:1, 16:9, 9:16, 4:3 など）デフォルト: 16:9
- `--output`: 出力ディレクトリ デフォルト: ./generated_images
- `--reference`: 参照画像のパス（複数指定可能、最大14枚）

### 使用例

基本的な画像生成：
```bash
$CCSKILL_NANOBANANA_DIR/venv/bin/python $CCSKILL_NANOBANANA_DIR/generate_image.py "猫がピアノを弾いている水彩画"
```

高解像度でワイド画像を生成：
```bash
$CCSKILL_NANOBANANA_DIR/venv/bin/python $CCSKILL_NANOBANANA_DIR/generate_image.py "夕焼けの海岸線" --resolution 4K --aspect 16:9
```

特定のディレクトリに出力：
```bash
$CCSKILL_NANOBANANA_DIR/venv/bin/python $CCSKILL_NANOBANANA_DIR/generate_image.py "ロゴデザイン" --output ./assets/images
```

### 参照画像を使った編集

既存の画像を参照して編集・変更ができます：

```bash
# 背景を変更
$CCSKILL_NANOBANANA_DIR/venv/bin/python $CCSKILL_NANOBANANA_DIR/generate_image.py "背景を夕焼けに変更して" --reference ./original.png

# 複数の参照画像を使用（ポーズ、スタイルなど）
$CCSKILL_NANOBANANA_DIR/venv/bin/python $CCSKILL_NANOBANANA_DIR/generate_image.py "この人物をこのポーズで描いて" --reference ./person.png --reference ./pose.png
```

参照画像の用途：
- 画像の部分編集（背景変更、色調整など）
- スタイル転送（別の画像のスタイルを適用）
- キャラクター一貫性の維持（同じ人物を異なるシーンで）
- 複数画像の合成

## プロンプトガイド

### クイックリファレンス

> 出典: [7 tips to get the most out of Nano Banana Pro](https://blog.google/products/gemini/prompting-tips-nano-banana-pro/) - Google公式ブログ

プロンプトには以下の要素を含めると、より良い結果が得られます：

- **Subject（主題）**: 誰が/何が画像にいるか？具体的に。（例：光る青い目を持つ無表情なロボットバリスタ、小さな魔法使いの帽子をかぶったふわふわの三毛猫）
- **Composition（構図）**: ショットはどうフレーミングされているか？（例：超クローズアップ、ワイドショット、ローアングルショット、ポートレート）
- **Action（アクション）**: 何が起きているか？（例：コーヒーを淹れている、魔法を唱えている、野原を走っている途中）
- **Location（場所）**: シーンはどこで行われているか？（例：火星の未来的なカフェ、散らかった錬金術師の書斎、ゴールデンアワーの陽光に満ちた草原）
- **Style（スタイル）**: 全体的な美学は？（例：3Dアニメーション、フィルムノワール、水彩画、フォトリアリスティック、1990年代の製品写真）
- **Editing Instructions（編集指示）**: 既存画像を編集する場合は、直接的かつ具体的に。（例：男性のネクタイを緑に変更、背景の車を削除）

プロフェッショナルな結果を得るには、より具体的な指示を含めます：

- **構図とアスペクト比**: キャンバスを定義（例：「9:16の縦型ポスター」「シネマティックな21:9のワイドショット」）
- **カメラとライティング**: 撮影監督のようにショットを指示（例：「浅い被写界深度(f/1.8)のローアングルショット」「長い影を作るゴールデンアワーのバックライト」）
- **テキスト統合**: テキストの見た目と配置を明確に（例：「'URBAN EXPLORER'という見出しを、上部に太字・白・サンセリフフォントで配置」）

#### プロンプティングの黄金ルール

Nano-Banana Proは「思考する」モデルです。単にキーワードをマッチングするのではなく、意図、物理法則、構図を理解します。最高の結果を得るには、「タグの羅列」（例：犬、公園、4k、リアル）をやめて、クリエイティブディレクターのように振る舞いましょう。

**1. 再生成せずに編集する**

モデルは会話的な編集を理解することに非常に優れています。画像が80%正しければ、ゼロから新しいものを生成しないでください。代わりに、必要な具体的な変更を依頼するだけです。

例：「いいですね、でもライティングを夕焼けに変えて、テキストをネオンブルーにしてください。」

**2. 自然言語と完全な文を使う**

人間のアーティストにブリーフィングするようにモデルと話してください。適切な文法と説明的な形容詞を使いましょう。

- 悪い例：「クールな車、ネオン、街、夜、8k。」
- 良い例：「夜の雨の東京の街を疾走する未来的なスポーツカーのシネマティックなワイドショット。ネオンサインが濡れた舗装路と車の金属製シャーシに反射している。」

**3. 具体的で詳細に**

曖昧なプロンプトは一般的な結果を生みます。被写体、設定、ライティング、雰囲気を定義してください。

- 被写体：「女性」ではなく「ヴィンテージのシャネル風スーツを着た洗練された年配の女性」と言う。
- 素材感：テクスチャを説明する。「マット仕上げ」「ブラッシュドスチール」「柔らかいベルベット」「しわくちゃの紙」。

**4. コンテキスト（「なぜ」または「誰のため」）を提供する**

モデルは「思考する」ため、コンテキストを与えることで論理的な芸術的判断ができるようになります。

例：「ブラジルの高級グルメ料理本用のサンドイッチの画像を作成してください。」（モデルはプロフェッショナルな盛り付け、浅い被写界深度、完璧なライティングを推論します）。

#### テキストレンダリング、インフォグラフィック、ビジュアル合成

Nano-Banana Proは、読みやすくスタイリッシュなテキストのレンダリングと、複雑な情報をビジュアル形式に合成する最先端の機能を持っています。

**ベストプラクティス：**
- 圧縮：密なテキストやPDFをビジュアルエイドに「圧縮」するよう依頼する。
- スタイル：「洗練されたエディトリアル」「技術図面」「手描きのホワイトボード」などの見た目を指定する。
- 引用：欲しいテキストを引用符で明確に指定する。

**プロンプト例：**

決算報告インフォグラフィック（データ取り込み）：
```
[Googleの最新決算報告PDFを入力]
「この決算報告の主要な財務ハイライトをまとめた、クリーンでモダンなインフォグラフィックを生成してください。'売上成長'と'純利益'のチャートを含め、CEOの重要な引用をスタイリッシュなプルクオートボックスでハイライトしてください。」
```

レトロ・インフォグラフィック：
```
「アメリカンダイナーの歴史についての1950年代風レトロなインフォグラフィックを作成してください。'フード'、'ジュークボックス'、'装飾'の明確なセクションを含めてください。すべてのテキストが読みやすく、時代に合ったスタイルになるようにしてください。」
```

技術図面：
```
「この建物を平面図、立面図、断面図で説明する正投影図面を作成してください。'北立面'と'メインエントランス'を技術的な建築フォントで明確にラベル付けしてください。フォーマット16:9。」
```

ホワイトボードサマリー（教育用）：
```
「'Transformer Neural Network Architecture'のコンセプトを、大学の講義に適した手描きのホワイトボード図としてまとめてください。EncoderブロックとDecoderブロックに異なる色のマーカーを使用し、'Self-Attention'と'Feed Forward'の読みやすいラベルを含めてください。」
```

#### キャラクター一貫性とバイラルサムネイル

Nano-Banana Proは最大14枚の参照画像をサポートします（高忠実度は6枚）。これにより「Identity Locking」が可能になります—特定の人物やキャラクターを顔の歪みなしに新しいシナリオに配置できます。

**ベストプラクティス：**
- Identity Locking：明示的に述べる：「画像1の人物の顔の特徴をまったく同じに保ってください。」
- 表情/アクション：アイデンティティを維持しながら、感情やポーズの変化を説明する。
- バイラル構成：被写体と大胆なグラフィックやテキストを一度に組み合わせる。

**プロンプト例：**

「バイラルサムネイル」（アイデンティティ + テキスト + グラフィック）：
```
「画像1の人物を使用してバイラルビデオサムネイルをデザインしてください。顔の一貫性：画像1の人物の顔の特徴はまったく同じに保ちますが、興奮して驚いた表情に変更してください。アクション：人物を左側に配置し、フレームの右側を指差すポーズにしてください。被写体：右側に美味しそうなアボカドトーストの高品質画像を配置してください。グラフィック：人物の指とトーストをつなぐ太い黄色の矢印を追加してください。テキスト：中央に巨大なポップスタイルのテキスト「3分で完成！」をオーバーレイしてください。太い白のアウトラインとドロップシャドウを使用してください。背景：ぼかした明るいキッチンの背景。高い彩度とコントラスト。」
```

「ふわふわフレンズ」シナリオ（グループ一貫性）：
```
[3つの異なるぬいぐるみクリーチャーの画像を入力]
「この3人のふわふわフレンズがトロピカルバケーションに行く面白い10パートストーリーを作成してください。ストーリーは全体を通してスリリングで、感情の高低があり、ハッピーな瞬間で終わります。3キャラクターすべての服装とアイデンティティを一貫させてください。ただし、表情とアングルは10枚の画像全体で変化させてください。各画像には各キャラクターを1人ずつだけ含めてください。」
```

ブランドアセット生成：
```
[商品の画像1枚を入力]
「受賞歴のあるファッションエディトリアルのような9枚の素晴らしいファッションショットを作成してください。このリファレンスをブランドスタイルとして使用しますが、プロフェッショナルなデザインタッチを伝えるようにニュアンスとバリエーションを加えてください。画像を1枚ずつ9枚生成してください。」
```

#### 高度な編集、復元、カラー化

モデルは会話的なプロンプトによる複雑な編集に優れています。これには「インペインティング」（オブジェクトの削除/追加）、「復元」（古い写真の修復）、「カラー化」（漫画/白黒写真）、「スタイルスワッピング」が含まれます。

**ベストプラクティス：**
- セマンティック指示：手動でマスクする必要はありません。変更したいことを自然に伝えるだけです。
- 物理理解：「このグラスに液体を入れて」のような複雑な変更を依頼して物理生成をテストできます。

**プロンプト例：**

オブジェクト削除とインペインティング：
```
「この写真の背景から観光客を削除し、周囲の環境に合った論理的なテクスチャ（石畳や店先）でスペースを埋めてください。」
```

漫画/コミックのカラー化：
```
[白黒の漫画パネルを入力]
「この漫画パネルをカラー化してください。鮮やかなアニメスタイルのパレットを使用してください。エネルギービームのライティング効果が輝くネオンブルーで、キャラクターの衣装が公式カラーと一致するようにしてください。」
```

ローカライゼーション（テキスト翻訳 + 文化的適応）：
```
[ロンドンのバス停広告の画像を入力]
「このコンセプトを東京の設定にローカライズしてください。タグラインを日本語に翻訳することも含めて。背景を夜の賑やかな渋谷の街に変更してください。」
```

ライティング/季節コントロール：
```
[夏の家の画像を入力]
「このシーンを冬に変えてください。家の建築はまったく同じに保ちますが、屋根と庭に雪を追加し、ライティングを寒くて曇った午後に変更してください。」
```

#### 次元変換（2Dから3D）

新しい強力な機能は、2Dの図面を3Dビジュアライゼーションに変換すること、またはその逆です。これはインテリアデザイナー、建築家、ミームクリエイターに最適です。

**プロンプト例：**

2D平面図から3Dインテリアデザインボード：
```
「アップロードされた2D平面図に基づいて、1枚の画像でプロフェッショナルなインテリアデザインプレゼンテーションボードを生成してください。レイアウト：上部に1つの大きなメイン画像（リビングエリアのワイドアングルパースペクティブ）、下に3つの小さな画像（マスターベッドルーム、ホームオフィス、3Dトップダウン平面図）のコラージュ。スタイル：すべての画像にわたって、温かみのあるオークの木目フローリングとオフホワイトの壁を持つモダンミニマリストスタイルを適用。品質：フォトリアリスティックレンダリング、ソフトな自然光。」
```

2Dから3Dミーム変換：
```
「'This is Fine'の犬のミームをフォトリアリスティックな3Dレンダーに変換してください。構成は同一に保ちますが、犬はぬいぐるみのように、炎はリアルな炎のように見えるようにしてください。」
```

#### ワンショットストーリーボードとコンセプトアート

グリッドなしで連続アートやストーリーボードを生成でき、単一セッションで一貫したナラティブフローを確保できます。これは「映画コンセプトアート」（例：今後の映画の偽リーク）でも人気があります。

**プロンプト例：**
```
「受賞歴のある高級ラゲージコマーシャルに登場する女性と男性をフィーチャーした、中毒性のある魅力的な9パートストーリーを9枚の画像で作成してください。ストーリーには感情の高低があり、ロゴと一緒に女性のエレガントなショットで終わります。女性と男性のアイデンティティと服装は全体を通して一貫している必要がありますが、異なるアングルと距離から見ることができ、そうすべきです。画像を1枚ずつ生成してください。すべての画像が16:9の横長フォーマットであることを確認してください。」
```

#### 構造制御とレイアウトガイダンス

入力画像はキャラクター参照や編集対象に限定されません。最終出力の構成とレイアウトを厳密に制御するために使用できます。これは、ナプキンスケッチ、ワイヤーフレーム、または特定のグリッドレイアウトを洗練されたアセットに変換する必要があるデザイナーにとってゲームチェンジャーです。

**ベストプラクティス：**
- ドラフトとスケッチ：手描きのスケッチをアップロードして、テキストとオブジェクトの配置を正確に定義する。
- ワイヤーフレーム：既存のレイアウトやワイヤーフレームのスクリーンショットを使用して、高忠実度のUIモックアップを生成する。
- グリッド：グリッド画像を使用して、タイルベースのゲームやLEDディスプレイ用のアセットを生成するようモデルに強制する。

**プロンプト例：**

スケッチから最終広告：
```
「このスケッチに従って[製品]の広告を作成してください。」
```

ワイヤーフレームからUIモックアップ：
```
「これらのガイドラインに従って[製品]のモックアップを作成してください。」
```

ピクセルアートとLEDディスプレイ：
```
「この64x64グリッド画像にぴったり収まるユニコーンのピクセルアートスプライトを生成してください。高コントラストの色を使用してください。」
（ヒント：開発者はその後、各セルの中心色をプログラムで抽出して、接続された64x64 LEDマトリクスディスプレイを駆動できます）。
```

スプライト：
```
「ドローンでバック宙をする女性のスプライトシート、3x3グリッド、シーケンス、コマ送りアニメーション、正方形のアスペクト比。添付の参照画像の構造に正確に従ってください。」
（ヒント：その後、各セルを抽出してgifを作成できます）
```

### 現在の制限事項

- 小さいテキストや細部の描画、正確なスペリングは完璧でない場合があります
- 図表やインフォグラフィックの事実的正確性は常に検証してください
- 多言語テキスト生成では文法ミスや文化的ニュアンスの欠落が起こる可能性があります
- 合成やライティング変更などの高度な編集では不自然なアーティファクトが発生することがあります

## 出力

- 画像は指定されたディレクトリ（デフォルト: `./generated_images`）に保存されます
- ファイル名はタイムスタンプ形式（例: `20251130_153045.png`、`20251130_153045.jpg`）
- 拡張子はAPIが返す画像の形式に応じて自動的に決定されます（PNG/JPEG/WebP）

## 注意事項

- 環境変数 `GEMINI_API_KEY` が設定されている必要があります
- Nano Banana Pro は有料APIのため、課金が発生します
- 生成された画像には SynthID ウォーターマークが含まれます
